模擬試験１

各問題の４つの選択肢から正解だと思うものを１つ選択してください。
答えが分からなかったり、間違えたりした箇所はリンクから各ページに飛んで復習してください。

問題1.
AWSクラウド上でIaaSを利用するメリットを選択してください。

1. オンデマンドによる柔軟性正解です。
オンデマンドで必要な時に必要な数だけサーバーなどのインフラを構築したり、削除したり、拡張したりといった対応が即座に柔軟に実施できることがクラウドの利点です。

2. インフラ管理の柔軟性2.は不正解です。
物理的なインフラ管理はユーザー側では実施できない場合が多いです。
そのためインフラ管理には柔軟性はなく、限定的な範囲での管理が求められています。

3. オンプレミス環境と比較して必ずコストが安くなる3.は不正解です。
コストは安くなることが多いですが、必ず安くなると確証があるものではありません。
重量課金でいろいろなサービスを使いすぎてしまった場合などに、余計なコストがかかる可能性があります。

4. セキュリティ対応のクラウドへの移管4.は不正解です。
セキュリティ対応は気にしなくてよいわけではありません。
自身が構築したアプリサービスに応じたセキュリティ対応をする必要があります。

問題2.
1週間だけ利用するキャンペーンサイトに対して選択するべきEC2インスタンスの購入オプションはどれでしょうか？

1. リザーブドインスタンス1.は不正解です。
リザーブドインスタンスは「１年間」「３年間」と期間を決めてEC2インスタンスを予約する事で通常のEC2インスタンスよりも格安にインスタンスを使うことができる購入です。
リザーブドインスタンスは長期利用を想定しているため、1週間だけ利用するインスタンスとしては不適切です。

2. スケジュールドリザーブドインスタンス2.は不正解です。
スケジュールドリザーブドインスタンスは日次、週次、月次と３パターンのスケジューリングされた買い方ができます。
月一回だけバッチ処理で利用したいなどの要望に利用できます。
中長期にわたって一定の利用量が求められるため、1週間だけ利用するインスタンスとしては不適切です。

3. スポットインスタンス3.は不正解です。
スポットインスタンスはオンデマンド価格より低価で利用できるAWSが管理する未使用の EC2 インスタンスです。
スポットインスタンスはユーザーが一時的に格安で利用することができますが、途中でAWS側で利用停止させられる可能性があります。
今回はキャンペーンサイトで１週間の利用ですが、スポットインスタンスはより短時間のスケーリングなどに利用するべきインスタンスです。
一週間の利用期間はスポットインスタンスとしては長すぎ、また途中で停止されるリスクもかなり高くなるため不適切です。

4. オンデマンドインスタンス正解です。
オンデマンドインスタンスは利用時間に応じて料金を支払う通常のインスタンス利用形態です。
今回は1週間だけインスタンスを利用するには中長期の予約はできないため、オンデマンドインスタンスを利用することが望ましいです。
また、このシナリオではスポットインスタンスを利用するには長すぎる期間であり、スポットインスタンスも利用できません。

次のうちAuroraデータベースの特性として正しい内容を選択してください。

問題7.
EBSボリュームのデータバックアップ方式として正しい内容を選択してください。

1. EBSのレプリカを作成する1.は不正解です。
EBSボリュームのバックアップを作成するには、スナップショット機能を使用する必要があります。
EBSではレプリカや自動バックアップは利用できません。

2. EBSボリュームの自動バックアップを設定する2.は不正解です。
EBSボリュームのバックアップを作成するには、スナップショット機能を使用する必要があります。
EBSではレプリカや自動バックアップは利用できません。

3. EBSに対してDMLを設定して、バックアップを構成する必要がある3.は不正解です。
DMLを利用することでスナップショット取得をスケジューリングすることができますが、バックアップ方式として必須の対応ではなく、オプショナルで実施する内容となっています。

4. EBSスナップショットを作成する正解です。
EBSボリュームのスナップショットを作成することで、EBSボリュームのバックアップを確保できます。
EBSスナップショットは増分バックアップです。
つまり、最初のスナップショットはすべてのデータのバックアップを取得しますが、二世代以降のスナップショットは最後のスナップショット後に変更されたデバイス上のブロックのみが保存されます。
これにより、スナップショットの作成に必要な時間を最小限に抑え、データを複製しないため、ストレージコストを節約できます。

問題8.
RDSに対して高速キャッシュ処理を実現したいと考えています。
どのサービスを利用するべきでしょうか？

1. Redshift1.は不正解です。
RedshiftはAWSで動作するデータウェアハウスです。
RedshiftはRDSと連携して高速処理を実行することはできません。

2. ElastiCache正解です。
Amazon ElastiCache を利用することで、RDSのクエリをキャッシュに保持して高速アクセスする連携を構築することができます。
Amazon ElastiCacheはオープンソースのRedisとMemcachedを利用したインメモリデータストア型の高速キャッシュデータベースサービスです。
このDBを利用して、高スループットかつ低レイテンシーなインメモリデータストアからデータを取得して、大量のデータを扱うアプリケーションを構築したり、既存のアプリケーションのパフォーマンスを改善したりすることが可能です。

3. S3 Transfer Acceleration3.は不正解です。
S3 Transfer AccelerationはS3データへの高速転送を可能にする機能です。
RDSのキャッシュ利用とは無関係です。

4. Aurora4.は不正解です。
AuroraはMySQL や PostgreSQL と互換性のあるリレーショナルデータベースです。
通常のRDSよりも高速なパフォーマンスが実現できますが、RDSと連携したデータ高速処理にはElastiCacheの方が向いています。

問題9.
コンポーネント間の通信を確実に実施するために、連携処理機能として利用するべきサービスを選択してください。

1. SES1.は不正解です。
Amazon SESはアプリケーションなどにEメール機能を実装可能にするEメールサービスであり、コンポーネント間の連携には利用されません。

2. API Gateway2.は不正解です。
Amazon API Gateway は簡単に API の作成、配布、保守、監視、保護が行えるフルマネージド型のAPI管理サービスです。
これもコンポーネント間の連携には利用されません。

3. Lambda3.は不正解です。
AWS Lambda を使用すれば、サーバーのプロビジョニングや管理なしでコードを実行できます。
これはコンポーネント間のメッセージ送受信に利用するサービスではありません。

4. SQS正解です。
Amazon SQSはメッセージキューを作成・管理するフルマネージド型サービスです。
SQSはソフトウェアコンポーネント間でメッセージを送受信でき、コンポーネント間のタスク連携や並列処理を実現することができます。
この仕組みにより、コンポーネントを切り離して独立して処理を実行できるため、システム全体のフォールトトレランスが向上します。
全てのキューメッセージは喪失しないように複数のアベイラビリティーゾーンにわたって冗長的に保存されます。

問題10.
あなたは開発中のアプリケーションの災害時のダウンタイムを防止したいと考えています。
どの展開方式を選択するべきでしょうか？

1. マルチAZでの展開1.は不正解です。
AWSグローバルインフラストラクチャは、リージョンとアベイラビリティーゾーン（AZ）を中心に構成されています。
各AWSリージョンは、個別の地理的なエリアとなっています。AWSリージョンは、更にアベイラビリティーゾーンと呼ばれる複数の分離されたロケーションによって構成されています。
マルチAZ構成にすることで一つのAZが停止しても、稼働を継続できる冗長性と可用性の高いシステムを構築することが可能ですが、災害などによってリージョン全体におよぶ障害が発生した場合には対応できません。
このシナリオでは、災害時のダウンタイムを予防することが求められており、リージョン障害にまで対応した構成が必要となります。
あなたは複数リージョン間にセカンダリー構成をスタンバイさせる構成を準備することで、冗長性と耐障害性をさらに高めることができます。
したがって、オプション２のマルチリージョンが正解です。

AWS Trusted Advisorを利用することで、現在利用しているAWSリソースに対するコスト最適化にむけた改善事項を確認することができます。

2. CloudWatchアラーム2.は不正解です。
CloudWatchアラームは何らかのイベントに対して通知設定することができる機能です。

3. AWS Config3.は不正解です。
AWS Configは、AWS リソースの設定を評価、監査、審査できるサービスです。
Config では、AWS リソースの設定が継続的にモニタリングおよび記録され、望まれる設定に対する記録された設定の評価を自動的に実行できます。
AWS ConfigにはAWSクラウドのコスト最適化を評価する機能はありません。

4. AWS Billing and Cost Management4.は不正解です。
AWS Billing and Cost Management は、AWS の請求書の支払い、使用量のモニタリング、コストの分析と管理に使用するサービスです。

問題64.
従来のサーバーの代わりにEC2インスタンスを使用する主な利点は何ですか？

1. フォールトトレラントが向上する正解です。
EC２インスタンスを利用することでオンプレミスサーバーにおいて複雑な設定が必要だったフォールトトレラントアプリケーションを容易に構築できます。
たとえば、ELBを設置して複数のインスタンスと複数のアベイラビリティーゾーン間でトラフィックを分散して、健全なEC2インスタンスのみがトラフィックを受信できるようにすることで、フォールトトレランスを改善できます。

2. データバックアップと保護が完全自動になる2.は不正解です。
データバックアップと保護が完全自動ではなく、ユーザーがデータ暗号化や手動でのスナップショット作成が実施することが必要です。

3. ソフトウェアに対するパッチ管理をAWS側が実行する3.は不正解です。
EC2インスタンスで利用しているソフトウェアに対するパッチ管理はユーザーが実施することが求められています。

4. クラウドへのリモートアクセスが向上する4.は不正解です。
クラウドへのリモートアクセスの向上というのは外部環境に依存されるものであり、EC2インスタンスの特徴ではありません。

問題65.
小規模なWEBサイトを構築するために簡易なサーバーセットを提供するサービスを選択してください。

1. Amazon Lightsail正解です。
小規模なWEBサイトやアプリケーションを構築するにはEC2インスタンスよりもAmazon Lightsailを使用することが最適です。
Lightsailはより単純なワークロード、迅速なデプロイメント、および AWS の使用開始に最適です。
Lightsailは小規模の仕様から利用を開始して拡張できるように設計されているVPSです。
シンプルなウェブアプリとアプリケーションを構築する際に利用するのに最適なサービスです。

2. Amazon EC22.は不正解です。
Amazon EC2は、安全でサイズ変更可能なコンピューティング性能をクラウド内で提供するウェブサービスです。
小規模なWEBサイトやアプリケーションを構築するにはEC2インスタンスよりもAmazon Lightsailを使用することが最適です。

3. Amazon ECS3.は不正解です。
Amazon Elastic Container Service (Amazon ECS) は、フルマネージド型のコンテナオーケストレーションサービスです。

4. Amazon EKS4.は不正解です。
Amazon Elastic Kubernetes Service (Amazon EKS) は、AWS クラウドまたはオンプレミスで Kubernetes アプリケーションを開始、実行、スケーリングする柔軟性を支援します。
