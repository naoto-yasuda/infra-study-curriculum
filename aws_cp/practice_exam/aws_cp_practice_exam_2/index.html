問題1.
NATゲートウェイの特徴について正しい内容を選択してください。

1. NATゲートウェイはインターネットからプライベートサブネットへとアクセスを可能にするため、プライベートサブネットに設置する1.は不正解です。
NATゲートウェイはパブリックサブネットに設置する必要があります。

2. NATゲートウェイはプライベートサブネットからインターネットへとアクセスを可能にするため、プライベートサブネットに設置する2.は不正解です。
NATゲートウェイはパブリックサブネットに設置する必要があります。

3. NATゲートウェイはプライベートサブネットからインターネットへとアクセスを可能にするため、パブリックサブネットに設置する正解です。
NATゲートウェイはプライベートサブネットからインターネットへの返信を可能にするため、パブリックサブネットに設置します。
その上で、プライベートサブネットのルートテーブルにNATゲートウェイへのルートを設定することが必要です。
これにより、NATゲートウェイはプライベートサブネットからのパブリックサブネットに送信されるトラフィックに対してアドレス変換処理をして、インターネットゲートウェイへとトラフィックを渡します。
これで、パブリックサブネットを通してインターネットへと返答できるようになります。

4. NATゲートウェイはインターネットからプライベートサブネットへとアクセスを可能にするため、パブリックサブネットに設置する4.は不正解です。
NATゲートウェイがあってもインターネットからプライベートサブネットへとアクセスは遮断されたままです。

問題8.
会社のセキュリティポリシーによりクラウドサーバーの物理領域を、自社内であっても他部署と共有することが不可能であるとされています。
どのような形式であればEC2インスタンスを利用してサーバーを活用することが可能でしょうか。

1. Dedicated Host正解です。
Dedicated hostはEC2インスタンスを物理的に占有する場合に選択するオプション設定です。
たとえ同じAWSアカウント内でも別のIAMグループやユーザーとは物理的なサーバーを共有することがありません。

2. ハードウェア専有インスタンス2.は不正解です。
ハードウェア専有インスタンスは、ユーザー専用のハードウェアの VPC で実行される Amazon EC2 インスタンスです。
これはホストハードウェアのレベルで、他の AWS アカウントに属するインスタンスから物理的に分離されています。
しかしながら、同じAWSアカウントであれば、他のIAMグループとインスタンスを共有する可能性があります。

3. ベアメタルサーバー3.は不正解です。
ベアメタルサーバーは物理サーバーに対してプロセッサーやストレージなどのリソースをユーザーがコントロールすることができます。

4. オンデマンド4.は不正解です。
オンデマンドインスタンスは一般的なクラウドのインスタンスの利用方法です。
これは他のユーザーとサーバーを共有します。

ELBによるトラフィック制御がマネージド型で提供されていることは需要変動による価格には影響がありません。

3. マルチAZ構成を実現できること3.は不正解です。
マルチAZ構成は需要変動に対する効果ではなく、障害発生時の可用性の確保として機能します。

4. CloudWatchによるモニタリングが無料であること4.は不正解です。
CloudWatchによるモニタリングが無料であることは、モニタリングコストを低減させることにつながりますが、需要変動に対する価格メリットにはなりません。

問題63.
EC2インスタンスの構成を別リージョンに再現したい場合にどのように対応する必要がありますか。

1. 対象インスタンスのAMIを別リージョンにコピーする正解です。
EC2インスタンスの構成を別リージョンに再現したい場合は、対象インスタンスのAMIを取得して別リージョンにコピーします。
その上で、別リージョンにおいてそのAMIを選択して新規にEC2インスタンスを起動することで同じ構成のEC2インスタンスを構築することができます。

2. 対象インスタンスのスナップショットを別リージョンにコピーする2.は不正解です。
対象インスタンスの内容はAMIでコピーすることが必要です。スナップショットはEBSをコピーする際に利用します。

3. 対象インスタンスを別リージョンに移行する3.は不正解です。
対象インスタンスそのものを別リージョンに移行することはできません。

4. クロスリージョンスナップショットを利用する4.は不正解です。
対象インスタンスの内容はAMIでコピーすることが必要です。スナップショットはEBSをコピーする際に利用します。

問題64.
S3を利用して静的WEBサイトを構成するために必要のない設定を選択してください。

1. 静的Webホスティングを有効化する1.は不正解です。
S3静的WEBホスティングを利用するためには静的ウェブサイトホスティングの有効化、インデックスドキュメントの設定、およびアクセス許可の設定を行う必要があります。

2. パブリックアクセスブロックを無効化する2.は不正解です。
静的ウェブサイトが閲覧可能にするためにパブリックアクセスブロックを無効化することが必要です。

3. バケットポリシーを設定する3.は不正解です。
HTMLファイルをパブリックに読み取り可能にするには、すべてのユーザーに s3:GetObject アクセス許可を付与するバケットポリシーを作成する必要があります。

4. アクセスコントロールポリシーを設定する正解です。
S3静的WEBホスティングを利用するためにはアクセスコントロールポリシーを設定するのではなく、バケットポリシーを利用して、すべてのユーザーに s3:GetObject アクセス許可を付与する必要があります。

問題65.
サブネットの特徴として間違っている内容を選択してください。

1. サブネットでプライベートネットワークを構成する1.は不正解です。
プライベートサブネットを利用してプライベートネットワーク空間を構成することができます。

2. ネットワークACLを適用する2.は不正解です。
サブネットにはネットワークACLを利用してトラフィック制御を実施します。

3. ルートテーブルを適用する3.は不正解です。
サブネットにはルートテーブルを適用して、ルーティング方法を設定します。

4. １つのサブネットで複数AZに跨いで設置することが出来る正解です。
１つのサブネットで１つのAZを指定して設置することが必要です。
