# EC2の削除
ハンズオンを終えたので作成したEC2インスタンスを削除しましょう。  
そのままインスタンスが起動したままになっていると課金が行われてしまうので注意してください。

## 1. EC2の管理コンソールの表示
![](../../aws_cp_delete_ec2_delete_ec2_1.png)
EC2の管理コンソールを表示します。

## 2. インスタンスの終了
![](../../aws_cp_delete_ec2_delete_ec2_2.png)
削除するEC2インスタンスを選択し、「インスタンスの状態」-「インスタンスを終了」をクリックします。  
なお「インスタンスを停止」は電源をオフにして削除はしません。  
今後また利用する可能性がある場合は終了ではなく停止を選んでも良いと思います。  
「インスタンスを再起動」は文字通りサーバーの再起動を行います。

![](../../aws_cp_delete_ec2_delete_ec2_5.png)
確認ウィンドウが表示されるので、改めて対象のインスタンスIDを確認して「終了」をクリックします。

## 3. インスタンスの終了確認
![](../../aws_cp_delete_ec2_delete_ec2_6.png)
「インスタンスの状態」が「シャットダウン中」から「終了済み」になることを確認します。  
「終了済み」になってもしばらくの間EC2の一覧には表示されますが、既に削除はされているので安心してください。  
一定の時間が経過すると一覧からも表示されなくなります。

## 補足. インスタンスの終了保護
インスタンスに終了保護が行われている場合はそのままでは終了を行うことができません。  
（デフォルトでは終了保護は無効になっています）

![](../../aws_cp_delete_ec2_delete_ec2_3.png)
その場合、「アクション」-「インスタンスの設定」-「終了保護を変更」をクリックします。

![](../../aws_cp_delete_ec2_delete_ec2_4.png)
「有効化」のチェックを外して、「保存」をクリックします。  
これで終了ができるようになります。  
EC2インスタンスの削除はポチポチと操作をするだけで行えてしまうため、本番環境など削除しては大きな問題になるサーバーに対して終了保護の設定を行い、単純な操作ミスなどで誤削除してしまうことを防ぐために終了保護は利用します。
