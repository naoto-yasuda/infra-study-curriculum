### 問題1. <br/>どのくらいの利用量になるか不明なサーバーを立ち上げる際に、利用するべきEC2インスタンスを選択してください。
<details><summary>1. リザーブドインスタンス</summary>1.は不正解です。<br/>リザーブドインスタンスは利用する期間が決まっていないと、利用することができません。</details><br/>
<details><summary>2. オンデマンドインスタンス</summary>正解です。<br/>どのくらいの利用量になるか不明なサーバーを立ち上げる場合はオンデマンドインスタンスを利用して、利用量に応じた課金をすることで柔軟に対応することが可能です。</details><br/>
<details><summary>3. スポットインスタンス</summary>3.は不正解です。<br/>スポットインスタンスは一時的な利用に限定して利用するため、不適切です。</details><br/>
<details><summary>4. スケジュールドリザーブドインスタンス</summary>4.は不正解です。<br/>スケジュールドリザーブドインスタンスは週1回24時間などの一定の利用時間が決まっている利用方法となります。</details>

### 問題9. <br/>多数のデータアクセスの発生するアプリケーション向けのWEBサーバーとしてEC2インスタンスを利用したいと考えています。<br/>最も高いI/O性能を実現するためのEBSボリュームを選択してください。
<details><summary>1. プロビジョンドIOPS</summary>正解です。<br/>プロビジョンドIOPSはレイテンシーの影響が大きいトランザクションワークロード向けに設計された極めてパフォーマンスの高いSSDボリュームです。<br/>スループットやIO性能が最も高いボリュームです。</details><br/>
<details><summary>2. スループット最適化HDD</summary>2.は不正解です。<br/>高いスループットを必要とするアクセス頻度の高いワークロード向けの低コストの HDD ボリュームです。</details><br/>
<details><summary>3. マグネティック</summary>3.は不正解です。<br/>EBSマグネティックボリュームはハードディスクドライブ(HDD)がベースになっており、データのアクセス頻度が低い場合か、またはパフォーマンスの一定性が最も重要でない場合に、比較的小さいデータセットのワークロードに使用できます。</details><br/>
<details><summary>4. 汎用SSD</summary>4.は不正解です。<br/>幅広いトランザクションワークロードに対応できる価格とパフォーマンスのバランスが取れた汎用SSDボリュームです。</details>

※このテーマは[EBSの概要](aws_cp_ebs_overview)で取り上げました。

### 問題10. <br/>EBSの暗号化の仕組みとして正しい内容を選択してください。
<details><summary>1. AWS KMSを利用する必要がある</summary>正解です。<br/>Amazon EBSはAWS KMSと連携してEBSボリュームを暗号化および復号します。</details><br/>
<details><summary>2. 暗号化を有効化することで実施できる</summary>2.は不正解です。<br/>AWS KMSを利用しないで、デフォルトの暗号化の仕組みを利用することはできません。</details><br/>
<details><summary>3. EC2インスタンスとの間で暗号化通信を有効化する</summary>3.は不正解です。<br/>EC2インスタンスとの間で暗号化通信を有効化するといった設定はありません。<br/>EC2インスタンスとの通信はAWS側で管理されています。</details><br/>
<details><summary>4. デフォルトで暗号化されている</summary>4.は不正解です。<br/>EBSはデフォルトで暗号化は実行されていません。</details>
