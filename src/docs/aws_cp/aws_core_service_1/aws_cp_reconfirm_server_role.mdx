# サーバーの役割の再確認
ハンズオンで作成したEC2がどのようなサーバーの役割を指定していたのかを再確認してみましょう。
## 1. 今回のサーバーの設定
![](../../aws_cp_reconfirm_server_role_server_setting_image.png)
今回のハンズオンでは以下のような形でサーバーを構成しています。
#### ハードウェア：EC2インスタンス
物理的な機器の代わりにEC2インスタンスを作成してハードウェアの位置付けで利用しました。
#### サーバーソフトウェア：AmazonLinuxおよびApache
OSにはAmazonLinuxを利用し、そのAmazonLinux上にApacheをインストールすることでWebサーバーの機能を持たせました。
## 2. 今回のサーバーの処理
![](../../aws_cp_reconfirm_server_role_process_image.png)
今回のハンズオンでは以下のような流れでサーバーの処理が行われました。
#### 1. ブラウザからEC2インスタンスのパブリックIPv4アドレスへHTTPのリクエストが行われる
#### 2. Webサーバー上のApacheがリクエストを受け取る
#### 3. Apacheからレスポンスとしてindex.htmlの内容を返す
#### 4. ブラウザで受け取ったレスポンスに応じてHelloWorldの表示をする
## 3. まとめ
今回のハンズオンで分かったようにサーバーには何らかの役割を持ったソフトウェアをインストールすることでサーバーはその機能を提供できるようになります。  
今回であればWebサーバーの役割を持ったApacheをインストールしたことで、サーバーはWebサーバーの機能を提供できるようになりWebページを表示することができるようになった、というような形です。
