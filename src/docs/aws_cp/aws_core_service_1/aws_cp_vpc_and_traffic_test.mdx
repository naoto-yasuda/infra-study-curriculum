# VPCとトラフィック制御に関する模擬試験

各問題の４つの選択肢から正解だと思うものを１つ選択してください。  
答えが分からなかったり、間違えたりした箇所はリンクから各ページに飛んで復習してください。
### 問題1. <br/>VPCの特徴として正しい内容を選択してください。
<details><summary>1. プライベートネットワーク空間ではインターネットへの出力ができない</summary>1.は不正解です。<br/>プライベートネットワーク空間であるプライベートサブネットであってもNATゲートウェイを介してインターネットへの出力が可能です。</details><br/>
<details><summary>2. VPC内に全てのAWSリソースを構成することが必要である</summary>2.は不正解です。<br/>VPC外に設置するAWSサービスもあります。</details><br/>
<details><summary>3. AWSクラウドをカーブアウト（切り出し）することができる</summary>正解です。<br/>Amazon Virtual Private Cloud (Amazon VPC) では、AWSクラウドの論理的に分離されたセクションをプロビジョニングし、ユーザー側で定義した仮想ネットワーク内のAWSリソースを起動することができます。<br/>つまり、AWS内に自分専用の領域をカーブアウトすることができます。</details><br/>
<details><summary>4. 無制限にVPCを構成可能である</summary>4.は不正解です。<br/>VPCはアカウントに対してデフォルトのクォータがリージョンあたり 5 VPC で、ニーズに応じてリージョンあたり 100 個の VPC を持つことができます。</details>

※このテーマは[VPCの概要](aws_cp_vpc_overview)で取り上げました。

### 問題2. <br/>VPCの設定で正しい内容を選択してください。
<details><summary>1. VPCはデフォルトで１つ設置される</summary>1.は不正解です。<br/>AWSアカウントを登録すると各リージョンにデフォルトVPCは１つずつ自動で設置されます。</details><br/>
<details><summary>2. VPCはリージョンを指定して設置する</summary>正解です。<br/>VPCはリージョンを指定して設置するため、AZはサブネットを作成する際にリージョン内のAZを複数設定することが可能です。</details><br/>
<details><summary>3. VPCはグローバルに設置する</summary>3.は不正解です。<br/>VPCはリージョンを指定して設置します。</details><br/>
<details><summary>4. VPCはグローバルに一意なCIDR範囲を設定する</summary>4.は不正解です。<br/>VPCはグローバルに一意なCIDR範囲を設定するのではなく、VPC内で重複しないように設定することになります。</details>

※このテーマは[VPCの概要](aws_cp_vpc_overview)で取り上げました。

### 問題3. <br/>サブネットの特徴として正しい内容を選択してください。
<details><summary>1. サブネットはAZを指定して設置する</summary>正解です。<br/>サブネットは１つのAZを指定して設置することになります。</details><br/>
<details><summary>2. サブネットはリージョンを指定して設置する</summary>2.は不正解です。<br/>サブネットは１つのAZを指定して設置することになります。</details><br/>
<details><summary>3. サブネットはリージョンで一意なCIDR範囲を設定する</summary>3.は不正解です。<br/>サブネットはVPC内でVPCが設定した範囲のCIDR内で、VPCと他のサブネットのCIDR範囲と重複がない範囲を設定することが必要です。</details><br/>
<details><summary>4. サブネットはAZで一意なCIDR範囲を設定する</summary>4.は不正解です。<br/>サブネットはVPC内でVPCが設定した範囲のCIDR内で、VPCと他のサブネットのCIDR範囲と重複がない範囲を設定することが必要です。</details>

※このテーマは[VPCの概要](aws_cp_vpc_overview)で取り上げました。

### 問題4. <br/>パブリックサブネットの特徴として間違っている内容を選択してください。
<details><summary>1. パブリックサブネットはデフォルトで構成されている</summary>1.は不正解です。<br/>デフォルトVPC内にパブリックサブネットはデフォルトで構成されています。</details><br/>
<details><summary>2. パブリックサブネットになっているサブネットに対して、インターネットゲートウェイへのルートを設定することでインターネット接続が可能となる</summary>正解です。<br/>サブネットのルートテーブルにおいてインターネットゲートウェイへのルートを設定することでインターネット接続が可能なパブリックサブネットとなります。<br/>つまり、パブリックサブネットであるためにはインターネットゲートウェイと接続されていることが要件であるため、パブリックサブネットに対してインターネットゲートウェイを追加することはできないことになります。</details><br/>
<details><summary>3. パブリックサブネット内のインスタンスにはインターネットから接続が可能となる</summary>3.は不正解です。<br/>パブリックサブネット内のインスタンスにはインターネットから接続が可能となります。</details><br/>
<details><summary>4. NATゲートウェイはパブリックサブネットに設置する</summary>4.は不正解です。<br/>NATゲートウェイはパブリックサブネットに設置した上で、プライベートサブネットのルートテーブルへと設定します。</details>

※このテーマは[VPCによるネットワーク構築（ハンズオン）](aws_cp_vpc_hands_on)で取り上げました。

### 問題5. <br/>プライベートサブネットの特徴として正しい内容を選択してください。
<details><summary>1. サブネットをコンソール画面で作成するとプライベートサブネットが生成される</summary>正解です。<br/>サブネットをVPCのコンソール画面で作成すると、まずはプライベートサブネットが生成されます。<br/>その後の設定でパブリックサブネットなどに構成することが可能です。</details><br/>
<details><summary>2. プライベートサブネットを指定して、プライベートサブネットを生成する</summary>2.は不正解です。<br/>サブネットをVPCのコンソール画面で作成すると、まずはプライベートサブネットが生成されます。<br/>その後の設定でパブリックサブネットなどに構成することが可能です。</details><br/>
<details><summary>3. サブネットにNATゲートウェイを設定するとプライベートサブネットになる</summary>3.は不正解です。<br/>サブネットをVPCのコンソール画面で作成すると、まずはプライベートサブネットが生成されます。<br/>その後の設定でパブリックサブネットなどに構成することが可能です。</details><br/>
<details><summary>4. サブネットにインターネットゲートウェイを設定するとプライベートサブネットになる</summary>4.は不正解です。<br/>サブネットをVPCのコンソール画面で作成すると、まずはプライベートサブネットが生成されます。<br/>その後の設定でパブリックサブネットなどに構成することが可能です。</details>

### 問題6. <br/>ネットワークACLの特徴として正しい内容を選択してください。
<details><summary>1. デフォルトではアクセス不可に設定されている</summary>1.は不正解です。<br/>ネットワークACLはデフォルトではすべてのアクセスが許可されています。</details><br/>
<details><summary>2. 許可設定のみを実施する</summary>2.は不正解です。<br/>ネットワークACLは許可と拒否を設定することができます。</details><br/>
<details><summary>3. インスタンスに適用することができる</summary>3.は不正解です。<br/>インスタンスに適用するのはセキュリティグループであり、ネットワークACLはできません。</details><br/>
<details><summary>4. サブネットに適用することができる</summary>正解です。<br/>ネットワークACLはサブネットに適用して、トラフィック制御を実施します。</details>

※このテーマは[セキュリティグループとネットワークACL](aws_cp_securitygroup_and_networkacl)で取り上げました。

### 問題7. <br/>異なるVPC間を接続するために利用する機能を選択してください。
<details><summary>1. VPCピアリング</summary>正解です。<br/>VPC ピアリング接続は、2 つの VPC 間でプライベートなトラフィックのルーティングを可能にするネットワーキング接続です。<br/>どちらの VPC のインスタンスも、同じネットワーク内に存在しているかのように、相互に通信できます。</details><br/>
<details><summary>2. Direct Connectゲートウェイ</summary>2.は不正解です。<br/>Direct Connectゲートウェイはリージョン間を接続するために利用します。</details><br/>
<details><summary>3. VPCエンドポイント</summary>3.は不正解です。<br/>VPCエンドポイントはVPC内からVPC外のリソースにアクセスする際に利用します。</details><br/>
<details><summary>4. AWS Managed VPN</summary>4.は不正解です。<br/>AWS Managed VPNはVPNを利用してオンプレミス環境と接続するために利用します。</details>

※このテーマは[VPCの概要](aws_cp_vpc_overview)で取り上げました。

### 問題8. <br/>セキュリティグループの特徴として正しい内容を選択してください。
<details><summary>1. 許可と拒否を設定する</summary>1.は不正解です。<br/>セキュリティグループは許可のみを設定することができます。</details><br/>
<details><summary>2. インバウンドとアウトバウンドの両方を設定する必要がある</summary>2.は不正解です。<br/>セキュリティグループはステートフルであるため、インバウンドを設定すればアウトバウンドは自動的にトラフィックが許可されます。</details><br/>
<details><summary>3. インスタンスのトラフィックを制御する</summary>正解です。<br/>１つ以上のインスタンスのトラフィックを制御するファイアウォールとして、セキュリティグループを使用します。</details><br/>
<details><summary>4. VPCセキュリティグループはVPCのトラフィックを制御する</summary>4.は不正解です。<br/>VPCセキュリティグループはVPC範囲が指定されていますが、インスタンスに適用されます。</details>

※このテーマは[セキュリティグループとネットワークACL](aws_cp_securitygroup_and_networkacl)で取り上げました。

<details><summary>2. インターネット経由で直接に接続することができる</summary>2.は不正解です。<br/>インターネット経由で直接に接続するのではなく、インターネット上にVPNを構成することが必要です。</details><br/>
<details><summary>3. VPCエンドポイント経由で接続する</summary>3.は不正解です。<br/>VPCエンドポイントはVPC内からVPC外のリソースにアクセスする際に利用します。</details><br/>
<details><summary>4. VPCピアリング経由で接続する</summary>4.は不正解です。<br/>VPC ピアリング接続は、2 つの VPC 間でプライベートなトラフィックのルーティングを可能にするネットワーキング接続です。</details>

※このテーマは[VPCの概要](aws_cp_vpc_overview)で取り上げました。

### 問題12. <br/>次のうち、ネットワークACLの特徴を選択してください。
<details><summary>1. 許可のみをIn/Outで指定する</summary>1.は不正解です。<br/>許可のみをIn/outで指定するのはセキュリティグループの特徴です。<br/>ネットワークACLは許可と拒否の両方を設定することができます。</details><br/>
<details><summary>2. 全てのルールを適用する</summary>2.は不正解です。<br/>全てのルールを適用するのはセキュリティグループの特徴です。</details><br/>

<details><summary>4. ステートフルであり戻りトラフィックの考慮がいらない</summary>4.は不正解です。<br/>ステートフルであり戻りトラフィックの考慮がいらないのはセキュリティグループの特徴です。</details>
