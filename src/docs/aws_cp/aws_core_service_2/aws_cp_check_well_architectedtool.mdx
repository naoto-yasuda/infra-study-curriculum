# Well-Architected Toolの確認
Well-Architected Frameworkを確認していく検証ツールであるWell-Architected Toolを実際に触ってみましょう。

## 1. Well-Architected Toolの利用方法
Well-Architected Toolは構築中または運用中のアプリケーションがベストプラクティスに沿っているかを検証することができるツールです。  
以下のような流れで検証を行うことができます。  
1. 構築しているアプリケーションをWell-Architected Toolに設定する
2. ５つの原則に関する質問に回答する
3. 改善点などが提示される
## 2. Well-Architected Toolの用語
Well-Architected Toolには独特の用語が使われていますので、それを理解して利用していきましょう。

| 用語 | 意味 |
|-------|-------|
| ワークロード | ウェブサイト、ECサイト、モバイルアプリのバックエンド、分析プラットフォームなどビジネス価値をもたらす一連のコンポーネントが特定する |
| マイルストーン | 設計、テスト、実稼働、本稼働の製品ライフサイクルを通して進歩するアーキテクチャでの重要な変更内容のこと |
| レンズ | ベストプラクティスに照らしてアーキテクチャを評価し、改善すべき分野を特定する一貫した方法 |
| 高リスクの問題（HRI） | ビジネスに重大な悪影響を及ぼす可能性があるアーキテクチャおよび運用上の問題 |
| 中リスクの問題（MRI） | その程度は HRI より低いビジネス上の問題 |
## 3. Well-Architected Toolの利用
### 3-1. Well-Architected Toolの管理コンソールの表示
![](../../aws_cp_check_well_architectedtool_setting_tool_1.png)
左上の「サービス」をクリックして、「管理とガバナンス」-「AWS Well-Architected Tool」をクリックします。

### 3-2. ワークロードの設定
![](../../aws_cp_check_well_architectedtool_setting_tool_2.png)
「ワークロードの定義」をクリックします。

![](../../aws_cp_check_well_architectedtool_setting_tool_3.png)
以下のようにワークロードの設定を入力して、「次へ」をクリックします。

| 項目 | 設定値 | 概要 |
|-------|-------|-------|
| 名前 | test | ワークロードの名前<br/>ワークロードはビジネス価値をもたらす一連のコンポーネントなので基本的にはアプリケーションやシステム単位のものになるので、それを考慮した上で名前を付けましょう |
| 説明 | test | ワークロードの範囲と用途を示す簡単な説明<br/>名前と同じくワークロードの単位を考慮して記載しましょう |
| レビューの所有者 | testgroup | レビュープロセスを所有する個人またはグループの名前、Eメールアドレス |
| 環境 | 本番稼働前 | ワークロードを実行する環境の稼働状態 |
| リージョン | AWSリージョン - 東京リージョン | ワークロードを実行するリージョン<br/>AWS以外の環境も利用している場合は「AWS以外の領域」にチェックを入れて情報を入力する |
| アカウントID | 空欄 | ワークロードがまたぐ範囲のAWSアカウントのID<br/>AWSアカウントのIDは上部メニューのアカウント名をクリックすることで表示される「マイアカウント」の右にある数字<br/>複数アカウントにまたがっている環境なら複数IDを入力する |
| アーキテクチャ設計 | 空欄 | 設計情報へのリンクURL |
| 業界の種類 | デジタル広告 | どういった業界のアプリケーションか |
| 業界 | モバイル広告 | どういった業界のアプリケーションかの詳細 |

![](../../aws_cp_check_well_architectedtool_setting_tool_4.png)
どういった改善点を出すかを決めるレンズを設定します。  
サーバレスのサービスを利用している場合は「Serverless Lens」を選択するなど、アプリケーションの構成によって選択が変わりますが、今回は「Well-Architected Framework」だけを選択して「ワークロードの定義」をクリックしましょう。

### 3-3. レビューの実施
![](../../aws_cp_check_well_architectedtool_setting_tool_5.png)
ワークロードの設定が表示されるのでそれぞれの項目を見ていきます。  
まずは「マイルストーン」タブをクリックします。  
ここではアプリケーションの変更点がある場合、その変更点をマイルストーンとしてここで設定していきます。

![](../../aws_cp_check_well_architectedtool_setting_tool_6.png)
「プロパティ」タブをクリックします。
ここではワークロードを作成した際に設定した内容が確認できます。

![](../../aws_cp_check_well_architectedtool_setting_tool_7.png)
「共有」タブをクリックします。
ここでは他のアカウントと共有を行いたい場合、ここで設定することができます。

![](../../aws_cp_check_well_architectedtool_setting_tool_8.png)
「概要」タブをクリックします。  
ここではワークロードに関する質問の回答状況とそれを基にしたレビューの内容が表示されます。

![](../../aws_cp_check_well_architectedtool_setting_tool_9.png)
「レビューを開始」をクリックして、「AWS Well-Architected Framework」をクリックしてワークロードに関する質問に回答してレビューを進めていきます。

![](../../aws_cp_check_well_architectedtool_setting_tool_10.png)
ワークロードに関する質問が表示されるので1つずつ回答を行って「次へ」をクリックします。  
ただこの質問はかなり数が多く時間が掛かりますので今回は選択肢の中から適当に選んでもらってドンドン回答を進めましょう。  
当然実際に利用する場合は各質問に対して自分たちのアプリケーションやシステムはどの選択しに当てはまるのかをちゃんと考えた上で回答をします。

![](../../aws_cp_check_well_architectedtool_setting_tool_11.png)
質問に回答し終えると「回答済みの質問」の数字が更新され、その下の「リスク」にレビュー結果が表示されます。

![](../../aws_cp_check_well_architectedtool_setting_tool_12.png)
![](../../aws_cp_check_well_architectedtool_setting_tool_13.png)
レビュー結果の詳細は「レンズ」の「AWS Well-Architected Framework」をクリックすることで確認することができます。
