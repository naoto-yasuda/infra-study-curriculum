# クラウドの基礎
## 1. クラウドとは
クラウドとは、必要に応じて他社所有のハードウェアやソフトウェア等をネットワークを介して利用するシステム利用形態、のことを言います。  
例えば下記のようなケースではクラウドが活用されています。
- 短期的なイベント、キャンペーンのために一時的にサーバーが欲しい…  
- システムの需要が増えたらサーバーを増やして対応したいけど、需要が減ったらサーバーは減らしたい…  
- 今すぐデータベースが欲しいけどサーバーを調達したり、データベースを構築している時間が無い…  

この様に柔軟にサーバーやシステムを利用したい時にクラウドは便利な技術となります。
![](../../aws_cp_cloud_basics_cloud_image.png)
雲の上にあるような実体を意識する必要ないところからインターネットを介して色々なサービスを利用できる、という意味から
#### 雲＝Cloud（クラウド）の上にあるものを利用する　→　クラウド
と呼ばれています。
## 2. クラウドの実体
クラウドサービスも元を辿れば当然実体となるサーバーがあります。  
その実体となるサーバーで活用されているのが前回学んだ仮想化技術です。  
大量の仮想サーバーを用意しておくことでインターネットを通じて利用しに来るユーザーにそれぞれ必要な分だけのサービスを提供することができます。
## 3. クラウドの形態
![](../../aws_cp_cloud_basics_cloud_component.png)
クラウドには様々な形態があり、主に
- アプリケーションをクラウド化したソフトウェア
- ミドルウェアやOSをクラウド化したサービス
- サーバーやスマホ端末、ネットワーク回線などをクラウド経由で利用するサービス

などがあります。
## 4. クラウドの特徴
クラウドの特徴として「５つの基本特性」「３つのサービス」「３つの提供形態」があります。
### 4-1. クラウドの５つの基本特性
| ５つの基本特性               | 概要                                                                                                                                      |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| オンデマンド・セルフサービス | ユーザは人を介さずに必要に応じてサーバーやネットワーク、ストレージを設置・拡張・設定が可能です。                                          |
| 幅広いネットワークアクセス   | ネットワークを通してPC、スマホ、タブレットなど各種デバイスから利用することが可能です。                                                    |
| リソースの共有               | ハードウェアの使用容量などのリソースは複数のユーザで共有し、ユーザの需要に応じて動的に割り当てられます。<br/>物理的配置は考慮が不要です。 |
| 迅速な拡張性                 | ハードウェアなどの資源は必要に応じて自動または手動で増やしたり、減らしたりすることができます。                                            |
| サービスは計測可能           | 稼働状況が常に計測されており、利用状況をコントロール・最適化できます<br/>計測結果に応じて従量的（使った分だけ）な課金が可能です。         |
### 4-2. クラウドの３つのサービス形態
![](../../aws_cp_cloud_basics_cloud_service_form.png)
「3. クラウドの形態」で説明したようにクラウドには主に３つの形態があります。  
形態ごとにどこの範囲まではサービス提供するのかを表す用語として以下の３つの用語があります。

| 用語 | 読み方               | 概要                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| ---- | -------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| SaaS | サース               | Software as a Service（サービスとしてのソフトウェア）の略。<br/>クラウド経由でアプリケーションを利用できるサービスです。<br/>やりたいことをすぐに実現できることがメリットですが、既にあるアプリケーションを利用する形のため自由度が低いです。<br/>代表的なサービスとしてはGmailが挙げられます。                                                                                                                                                            |
| PaaS | パース               | Platform as a Service(サービスとしてのプラットフォーム)の略。<br/>クラウド経由でアプリケーションが稼働するためのデータベースやプログラム実行環境のプラットフォームを利用できるサービスです。<br/>インフラを構築する手間が省けプログラムだけを用意すれば良いのがメリットですが、データベースの設定やプログラムの実行環境に一定の制限があるので自由度はインフラから構築するのに比べ下がります。<br/>代表的なサービスとしてはGoogleAppsEngineが挙げられます。 |
| IaaS | イアース、アイアース | Infra as a Service（サービスとしてのインフラ）の略う。<br/>クラウド経由で仮想サーバーやハードディスクなどのインフラをクラウド経由で利用できるサービスです。<br/>SaaSやPaaSに比べ自由度が高いですが、その分インフラから構築する必要があるためインフラに関する知識を求められます。                                                                                                                                                                           |
### 4-3. クラウドの３つの提供形態
クラウド提供形態はプライベートとパブリックが主流です。  
加えて両方を併用するハイブリッド型の提供形態があります。  
クラウド以前の形態であるオンプレミスと合わせて説明します。

| 提供形態             | 所有形態 | 特徴                                                                                                                                                                                                                                                                                                                                                                             |
| -------------------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| オンプレミス         | 自社所有 | ハードウェア・ソフトウェアを購入し、自社にシステムを構築します。<br/>自社でハードウェアを自由にできるのがメリットですが、調達や構築に時間を要したり、運用管理に人員やコストを必要とするのがデメリットです。                                                                                                                                                                      |
| プライベートクラウド | 自社所有 | 自社内にクラウド基盤を構築して、ハードウェアやソフトウェアを事業部や子会社感で共同利用します。<br/>自社のサーバーなどの資産を効率的に利用でき、パブリッククラウドと比較して自由に利用できるのがメリットです。<br/>ただしオンプレミスと同様に自社で物理サーバーの管理が必要などクラウドの運用管理が難しい部分があり、クラウドを利用するメリットが低下ししまうのがデメリットです。 |
| パブリッククラウド   | 他社所有 | ハードウェアやソフトウェアを他社と共同利用し、柔軟に構築・解除することができます。<br/>運用管理を他社に任せられ、調達や構築がすぐにできたり、すぐに使わなくできるなど自由度が高いのがメリットです。<br/>ただし自社でコントロールできる範囲が制限されることがデメリットになります。                                                                                               |
| ハイブリッドクラウド | 両方     | オンプレミスやプライベートクラウド、パブリッククラウドを組み合わせて利用します。<br/>各形態のメリットを活かしつつデメリットをカバーできるのがハイブリッドクラウドのメリットになります。<br/>ただし色々な形態を組み合わせることになるためシステム構成は複雑化し、管理する対象も増えるため運用管理が難しくなることがデメリットです。                                               |
### 4-4. クラウドへの進化
![](../../aws_cp_cloud_basics_cloud_evolution.png)
元々「集中コンピューティング」という形があったところから更に処理速度を早くするために分散コンピューティングという方式が利用されるようになりました。  
その後2000年になってより柔軟に管理できるクラウドコンピューティングという方式が利用され始めました。  
近年はIoTなどの技術の発展に合わせてより細かい単位で処理が行われるような方式へ発展していっています。
### 4-5. クラウドサービスで実現できる様々な技術
#### 4-5-1. クラウドIoT
![](../../aws_cp_cloud_basics_cloud_iot.png)
クラウドサービスではIoTに関する機能が多数提供されていますが、中でもAWSは特に機能が豊富でグローバルのシェア率も高いです。
#### 4-5-2. クラウドAI
完成したAI機能をクラウドを通じて即座に利用することもできます。  
MicrosoftAzureやIBM Watson、GoogleMLなど多くのクラウドサービスで提供がされています。  
AWSでも機械学習や画像識別、CMなどでよく知られるAmazonAlexaのような音声コンシェルジュに関するサービスが提供されています。
### 4-5. クラウドファーストの時代
かつては自社でサーバーを持つことが当たり前だったので「クラウドはセキュリティ的にまずそう…」という意見もありましたが、今はインフラを構築するにあたってまずクラウドを考えるという「クラウドファースト」の時代になっています。  
年々ビジネスの環境変化の速さはドンドン加速しており、それに対応するインフラを作っていくためにはクラウドのスピード感が必要不可欠となっています。  
また私達の生活の各所にクラウドサービスが利用されており、自分のPCやスマホからクラウドサービスに常にアクセスしているような生活スタイルに変化しています。  
![](../../aws_cp_cloud_basics_net_and_real.png)
そしてIoTやウェアラブル端末などにより現実世界の行動データがクラウドに集約されたり、クラウド経由で各種端末を操作するなどクラウドによりネットと現実が融合した世界が既に進んでいます。
#### 4-5-1. クラウドのスピード感が求められる理由
ビジネスの環境変化の速さがドンドン加速していると言いましたがそれは以下の理由からです。
![](../../aws_cp_cloud_basics_cloud_spead_reason.png)
技術が進化することでその技術を使ったビジネスのスピードが加速し、  
また多様化していく顧客のニーズに全て対応するためにも変化の速さに対応できるクラウドが求められています。
