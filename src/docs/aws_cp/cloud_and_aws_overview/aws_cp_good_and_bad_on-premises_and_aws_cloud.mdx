# オンプレミスとAWSクラウドの良し悪し
## 1. クラウドのメリット／デメリット
オンプレミスと比較した時のクラウドのメリットは以下のとおりです。

| メリット | デメリット |
|-------|-------|
| オンデマンドでリソースを柔軟に迅速に活用することができる。<br/><br/>オンプレミス環境とのハイブリッドやオンプレミス環境にAWSクラウドサービスを展開することも可能。（項番３参照）<br/><br/>下手に自社DC内でインフラを構成するよりも、AWSに依存した方がよっぽどセキュリティは高い。 | クラウド業者のサービスに依存してしまうベンダーロックインが起こる。（項番２参照）<br/><br/>データ保存先や活用で社内での保管が義務付けられている等、クラウド環境が利用できないケースがある。<br/>→回避策がある（項番３参照） |
## 2. ベンダーロックインについて
オンプレミスのサーバーでシステムを構築して、オンプレミスのサーバーに移行するのであれば、サーバー変更やソフトウェアの変更をすることは比較的簡単に行なえます。
![](../../aws_cp_good_and_bad_on-premises_and_aws_cloud_vendor_lockin_onpre_to_onpre.png)

またEC2インスタンスなどオンプレミスのサーバーに近いサービスであれば、同様に移行をしてサーバー変更やソフトウェアの変更をすることは比較的簡単に行うことができます。
![](../../aws_cp_good_and_bad_on-premises_and_aws_cloud_vendor_lockin_ec2_to_onpre.png)

ただしAWS独自の機能をアプリケーションに組み込んで機能開発をしてしまうとAWS以外の環境に移行することが困難になります。  
例えばAWSの中でも利用されることの多いサーバーレスの仕組みはその代表例です。
![](../../aws_cp_good_and_bad_on-premises_and_aws_cloud_vendor_lockin_serverless_to_onpre.png)
## 3. オンプレミス環境のAWS
AWS Outpostsというサービスではオンプレミス環境にAWSサービスをセットアップして独自に利用できるように構築することができます。
