# AWSの活用メリット
## 1. IaaSとしての活用
![](../../aws_cp_cloud_basics_cloud_service_form.png)
AWSはSaaS、PaaS、IaaSそれぞれのサービスが存在していますが、AWSの基本となるのはIaaSでのサービスとなります。  
様々なインフラをインターネットを介してオンデマンドで利用できることがメリットです。  
企業によっては下記のように全てのインフラ・アプリをAWSに移行してそのメリットを最大限活かしている企業もあります。
##### 移行前：自社でオンプレミスを中心にしたインフラを抱えており、その運用は他社に任している
#### 　　　　　　　　　　　　　　　　　　↓
##### 移行後：全てのインフラをAWSに移行し、自社での開発・運用体制を整備した
## 2. PaaSとしての活用
AWSのIaaSに加え、アプリケーション開発ツール（PaaS）を利用して、アプリケーションを構築することができます。

| サービス名 | 概要 |
|-------|-------|
| コーディングツール | Cloud9 |
| コード管理<br/>デプロイ支援ツール | Codeシリーズ<br/>ElasticBeanStalk<br/>Git など |
| 開発環境のインフラ | EC2<br/>RDS<br/>VPC など |
例えば開発環境をEC2で用意し、コーディングはCloud9を活用、書いたコードはCodeシリーズを活用して管理やデプロイをすることでスピーディな開発を行うことができます。
## 3. SaaSとしての活用
AI翻訳機能や文章解析AIなどのAWSによって既に完成されたサービスをSaaSのサービスとして利用することができます。  
既に出来上がっているクラウドサービスをAPIを利用して組み込むことですぐに最先端の機能を利用することができます。
