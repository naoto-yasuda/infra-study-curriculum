# アプリケーション統合
## 1. アプリケーション統合に関するサービス
AWSにはアプリケーション間の通信制御やフロー作成・データ連携などに利用できるサービスとして以下のようなものがあります。

| サービス名 | 説明 |
|-------|-------|
| SNS | 完全マネージド型 pub/sub（受発信） メッセージング |
| SQS | 完全マネージド型のメッセージキューイングサービス |
| SES | Eメール送受信サービス |
## 2. Amazon Simple Notification Service
### 2-1. Amazon Simple Notification Serviceとは
![](../../aws_cp_application_integration_sns_image.png)
Amazon Simple Notification Service（Amazon SNS）とは、フルマネージド型のプッシュ型通知サービスで他のサービスとの非同期通信を可能にする事ができます。

![](../../aws_cp_application_integration_sns_topic_image.png)
送信側がトピックを作成して受信側をポリシー指定することで制御された非同期通信を実現します。
### 2-2. SNSの特徴
SNSには以下のような特徴があり、AWSの様々なサービスと連携して通知可能で、疎結合アーキテクチャに利用できます。
- 単一発行メッセージ
- メッセージ通信順番は保証されない
- 取り消し不可
- 配信ポリシーによる再試行を実施
- メッセージサイズは最大256KB
## 3. Amazon Simple Queue Service
### 3-1. Amazon Simple Queue Serviceとは
![](../../aws_cp_application_integration_sqs_image.png)
Amazon Simple Queue Service（Amazon SQS）とは、キューを溜め込んでポーリング処理を実施することが出来るサービスです。
### 3-2. ポーリングとは
![](../../aws_cp_application_integration_polling_image.png)
ポーリングとは一旦中継所に通信内容を貯めておいて、受信側のタイミングが良い時に通信を行うことが出来ます。
### 3-3. SQSの特徴
SQSには以下のような特徴があり、フルマネージド型で提供され、高可用性・高スケーラビリティ・高スループット・低コストを実現することが出来ます。
- 複数のサーバー／データセンターにメッセージを保持する高可用性構成
- 多数の送信者と受信者に対応可能なスケーラビリティの確保
- メッセージが増加しても高スループットを維持できる
- 無料枠と従量課金による低コスト
## 4. Amazon Simple Email Service
### 4-1. Amazon Simple Email Serviceとは
Amazon Simple Email Service（Amazon SES）とは、フルマネージド型／サーバレス型のコスト効率に優れたEメールサービスです。
### 4-2. SESの特徴
SESには以下のような特徴があります。
- スケーラブルな構成で信頼性が高いマネージド型
- メール送信 ：トランザクションメールなどの高品質なコンテンツを顧客に送信可能
- メール受信：受信したメールをトリガーにS3やLambdaなどを起動可能
- バウンス処理：メールが送れなかった場合の処理を規定
## 5. ユースケース分け
各サービスのユースケースに応じた使い分けを理解して、実際の試験問題に対応できるようにしましょう。
| サービス名 | 説明 |
|-------|-------|
| SNS | AWS上でイベント通知やメッセージング処理／プッシュ通知をするといった場合はSNS |
| SQS | AWS上でキューイング処理／タスク並列分散処理／ポーリング・Pull型の通知をする場合はSQS |
| SES | Eメール送受信機能を利用する場合はSES |
